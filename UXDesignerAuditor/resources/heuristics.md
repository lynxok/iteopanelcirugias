# Heurísticas de Diseño y Usabilidad

Como Senior Designer, utilizas estas reglas no como leyes rígidas, sino como fundamentos para mejorar la experiencia del usuario.

## 1. Visibilidad del Estado del Sistema
*   ¿El usuario sabe qué está pasando? (Loaders, skeletons, barras de progreso).
*   ¿Hay feedback claro después de una acción?

## 2. Relación entre el Sistema y el Mundo Real
*   Uso de lenguaje que el usuario entienda (evitar tecnicismos).
*   Iconografía intuitiva y familiar.

## 3. Jerarquía Visual y Arquitectura de Información
*   **Regla de los 3 niveles**: El ojo debe identificar primero el título, luego la acción principal y finalmente los detalles secundarios.
*   **Espaciado Negativo**: El espacio "blanco" no es espacio perdido, es claridad.

## 4. Prevención de Errores y Recuperación
*   Confirmaciones antes de acciones destructivas (Borrar, Salir sin guardar).
*   Mensajes de error que expliquen *cómo* solucionarlo, no solo que falló.

## 5. Carga Cognitiva
*   **Ley de Hick**: El tiempo que toma tomar una decisión aumenta con el número y la complejidad de las opciones. Minimiza las opciones por pantalla.
*   **Proximalidad**: Elementos relacionados deben estar visualmente cerca.
