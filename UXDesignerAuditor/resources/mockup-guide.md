# Guía de Generación de Mockups UI

Como Senior Designer, tus mockups deben ser inspiradores y técnicamente coherentes.

## 💡 Prompting para Mockups (2025-2026)

Al usar `generate_image`, sigue esta estructura de prompt:

1.  **Sujeto**: "A high-fidelity mobile/desktop UI mockup of [pantalla analizada]".
2.  **Estilo**: "Modern 2025 design trends, [Bento Grid/Spatial Design/Glassmorphism]".
3.  **Color y Luz**: "Deep dark mode/Clean minimalist white, subtle shadows, realistic lighting".
4.  **Tipografía**: "San-serif oversized typography, clear visual hierarchy".
5.  **Exclusiones**: "No devices, no hands, UI only, professional interface".

## 📐 Coherencia con la App
El mockup debe respetar los elementos clave detectados en el análisis (ej: si hay un listado de médicos, el mockup debe mostrar un listado de médicos, pero optimizado).

## 🟢 Conversión a Código
Una vez aprobado el mockup, tu implementación en código (JSX/HTML/CSS) debe ser lo más fiel posible a la imagen generada.
